# Норвежский новостной бот для Telegram

Каждое утро в 08:00 по Осло (летом; зимой — 07:00, см. примечание ниже) бот:

1. Собирает свежие новости (за последние 24 часа) с NRK, VG, Aftenposten и Dagbladet.
2. Выбирает до 5 самых свежих новостей, которые ещё не публиковались.
3. Переводит и адаптирует их на русский язык одним постом через Gemini.
4. Публикует пост в твой Telegram-канал со ссылками на оригиналы.
5. Запоминает, что уже опубликовано, чтобы не дублировать на следующий день.

## ⚠️ Сначала — смени старые ключи

Если токен и ключ из старой версии скрипта уже где-то "засветились" (например, были в публичном
репозитории), считай их скомпрометированными:

- **Telegram**: напиши [@BotFather](https://t.me/BotFather) → `/mybots` → выбери бота → `Revoke API Token`.
- **Gemini**: зайди в [Google AI Studio](https://aistudio.google.com/app/apikey), удали старый ключ,
  создай новый.

Новые значения нигде не публикуй и не пиши прямо в код — они идут только в GitHub Secrets (ниже).

## Шаг 1. Создай репозиторий на GitHub

1. Зайди на [github.com](https://github.com) → New repository.
2. Сделай его **приватным** (Private) — так безопаснее, хотя секреты всё равно не попадут в код
   даже в публичном репозитории, если всё настроено правильно.
3. Залей туда файлы из этой папки: `news_bot.py`, `requirements.txt`, `.gitignore`,
   `.github/workflows/news.yml`.

Если работаешь через сайт GitHub (без git в терминале) — можно просто перетащить файлы через
"Add file → Upload files", сохраняя структуру папок (`.github/workflows/news.yml` обязательно
должен лежать именно в этом пути).

## Шаг 2. Добавь секреты в репозиторий

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**.

Добавь три секрета (имена должны совпадать ровно так):

| Имя секрета      | Значение                                  |
|-------------------|--------------------------------------------|
| `TELEGRAM_TOKEN`  | Новый токен бота от BotFather              |
| `CHANNEL_ID`      | ID твоего канала, например `-1004421584028`|
| `GEMINI_KEY`      | Новый ключ из Google AI Studio             |

## Шаг 3. Проверь, что бот — админ канала

Бот должен быть добавлен в твой Telegram-канал как администратор с правом публикации сообщений.
Без этого `sendMessage` будет возвращать ошибку `Forbidden` или `chat not found`.

## Шаг 4. Запусти вручную для проверки

В репозитории: **Actions → Утренний новостной дайджест → Run workflow**.

Это запустит бота прямо сейчас, не дожидаясь утра — удобно для проверки, что всё работает.
Проверь логи запуска (там будет видно, какие новости найдены и отправились ли они) и сам канал.

После этого бот будет запускаться автоматически каждый день по расписанию из `news.yml`.

## О времени запуска (важно)

Cron-расписание в `news.yml` указано в UTC и не подстраивается под переход на летнее/зимнее
время автоматически. Сейчас (лето, UTC+2 в Осло) `cron: "0 6 * * *"` соответствует 08:00 по Осло.

Когда Норвегия перейдёт на зимнее время (конец октября, UTC+1), это же расписание будет
публиковать дайджест в 07:00 по Осло — на час раньше. Если хочешь, чтобы было ровно 08:00
круглый год, два раза в год меняй строку `cron: "0 6 * * *"` на `"0 7 * * *"` и обратно
вручную в файле `.github/workflows/news.yml`, либо позже добавим автоматическую проверку
времени внутри самого скрипта.

## Настройки, которые легко поменять

В начале `news_bot.py`:

- `TOP_N_NEWS` — сколько новостей включать в один пост (сейчас 5).
- `FRESHNESS_WINDOW_HOURS` — за какой период считать новости "свежими" (сейчас 24 часа).
- `RSS_SOURCES` — список источников; можно добавить ещё ленты или убрать существующие.
- `HISTORY_RETENTION_DAYS` — сколько дней помнить про уже опубликованные новости (сейчас 7).

## Локальный запуск (без GitHub, для отладки)

```bash
export TELEGRAM_TOKEN="..."
export CHANNEL_ID="-1004421584028"
export GEMINI_KEY="..."
pip install -r requirements.txt
python news_bot.py
```

## Известные ограничения

- RSS-ленты VG (`vg.no/rss/feed/?format=rss`), Aftenposten (`aftenposten.no/rss`) и Dagbladet
  (`dagbladet.no/rss/nyheter/`) найдены через открытые источники, но не протестированы вживую
  (нет доступа к интернету в среде, где собирался этот код). Структура их RSS может немного
  отличаться от NRK. Если при первом реальном запуске какой-то источник вернёт 0 новостей или
  ошибку, в логах будет видно, какой именно — тогда нужно будет проверить актуальный URL вручную
  на сайте источника (часто помогает поискать "site:vg.no rss" или посмотреть исходный код главной
  страницы на наличие `<link type="application/rss+xml">`).
- Если ни один источник не вернёт новостей (например, временно недоступны все сайты), бот просто
  завершится без публикации и попробует снова на следующий день — это нормально, но если такое
  повторяется несколько дней подряд, стоит проверить URL лент вручную.
